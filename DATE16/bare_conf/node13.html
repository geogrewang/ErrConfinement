Error Prediction Report

Token tracking analysis is carried out on an assembly testbench consisting of algorithmic and memory access instructions. Tokens are created at different hardware resources where the error prediction reports are documented in Table II. For instance the first group shows that the created token with ID 1 expands into totally tokens dynamically. Only 8 tokens live until the end of simulation while the rest ones have been removed or overwritten. Token in processor core is stored into data memory for memory access instruction. On the contrary the token in second group results in one error in data memory although just 3 tokens are expanded. In the last group, although 6 tokens are expanded, no token has been stored into data memory so that no application-level errors are visible. Based on the error prediction reports the user can easily perform vulnerability analysis for specific faults in the architecture.
